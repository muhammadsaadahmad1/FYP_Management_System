Deploy Firebase Security Rules

This page will help you deploy the updated Firebase security rules to fix the "Missing or insufficient permissions" error.

Step-by-Step Deployment

Step 1: Open Firebase Console

Go to the Firebase Console and select your project:

https://console.firebase.google.com/ Project: fypmanagementsystem-29faf

Step 2: Navigate to Firestore Rules

  1. In the Firebase Console, click on "Firestore Database" in the left menu
  2. Click on the "Rules" tab at the top
  3. You should see the current security rules

Step 3: Replace Security Rules

Copy the updated security rules below and paste them into the Firebase Console:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Users collection rules match /users/{userId} { // Allow read access for user lookup during login allow read: if true; // Allow write only if user is authenticated and owns the document allow write: if request.auth != null && request.auth.uid == userId; // Allow create if user is authenticated OR if creating during registration allow create: if request.auth != null || (request.resource.data.role == 'student' || request.resource.data.role == 'supervisor' || request.resource.data.role == 'admin'); // Allow update if user is authenticated and owns the document allow update: if request.auth != null && request.auth.uid == userId; // Allow delete if user is authenticated and owns the document allow delete: if request.auth != null && request.auth.uid == userId; } // Allow queries on users collection for login purposes match /users/{document} { allow read: if true; } // Groups collection rules match /groups/{groupId} { // Allow read/write if user is authenticated allow read, write: if request.auth != null; // Allow create if user is authenticated OR if creating during registration allow create: if request.auth != null || request.resource.data.groupSize != null; // Allow update if user is authenticated allow update: if request.auth != null; // Allow delete if user is authenticated allow delete: if request.auth != null; } // Projects collection rules match /projects/{projectId} { // Allow read/write if user is authenticated allow read, write: if request.auth != null; // Allow create if user is authenticated allow create: if request.auth != null; // Allow update if user is authenticated allow update: if request.auth != null; // Allow delete if user is authenticated allow delete: if request.auth != null; } // Proposals collection rules match /proposals/{proposalId} { // Allow read/write if user is authenticated allow read, write: if request.auth != null; // Allow create if user is authenticated allow create: if request.auth != null; // Allow update if user is authenticated allow update: if request.auth != null; // Allow delete if user is authenticated allow delete: if request.auth != null; } // Reports collection rules match /reports/{reportId} { // Allow read/write if user is authenticated allow read, write: if request.auth != null; // Allow create if user is authenticated allow create: if request.auth != null; // Allow update if user is authenticated allow update: if request.auth != null; // Allow delete if user is authenticated allow delete: if request.auth != null; } // Meetings collection rules match /meetings/{meetingId} { // Allow read/write if user is authenticated allow read, write: if request.auth != null; // Allow create if user is authenticated allow create: if request.auth != null; // Allow update if user is authenticated allow update: if request.auth != null; // Allow delete if user is authenticated allow delete: if request.auth != null; } // Test collection rules - for development and testing match /test_collection/{docId} { // Allow all operations for testing allow read, write, create, update, delete: if true; } // Public collection rules - for development match /public/{docId} { // Allow all operations for testing allow read, write, create, update, delete: if true; } // System collection rules - for system data match /system/{docId} { // Allow read for authenticated users allow read: if request.auth != null; // Allow write for authenticated users (admin only in production) allow write: if request.auth != null; } } }

Step 4: Publish Rules

  1. After pasting the rules, click the "Publish" button at the top
  2. Wait for the rules to be deployed (usually takes a few seconds)
  3. You should see a "Rules published successfully" message

Step 5: Test Registration

After deploying the rules, test the registration to verify the fix:

What Was Fixed

Security Rules Updated:

  • Users Collection: Now allows user creation during registration (unauthenticated users with valid role data)
  • Groups Collection: Now allows group creation during registration (unauthenticated users with groupSize data)
  • Other Collections: Still require authentication for security
  • Read Access: Still allows public read for login functionality

Security Considerations:

  • The rules allow registration operations but maintain security for other operations
  • Only users with valid role data (student/supervisor/admin) can be created
  • Only documents with groupSize data can be created in groups collection
  • All other operations still require authentication
